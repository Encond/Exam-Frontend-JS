Про проект - це сайт, і міні-гра в одному.

-Сайт: Має 3 сторінки {
	1. Play: Має в собі {
		1. Посібник для клавіатури, а також ON/OFF анімацій результат якої зберігається в localstorage,
		3. Міні-гра
	},
	2. Game History: Там де зберігаються попередні результати гри через localstorage з можливістю очищення,
	3. About: Інформація про гру
}

Анімації присутні на:
- Кнопках
- Тексті в посібнику до клавіатури
- Тексті в header елементі
- Самому header елементі (появлення із знизу екрана), разом з цим саме вікно міні-гри, і її елементи

Виключення анімації працює на:
- Самому header елементі (появлення із знизу екрана), разом з цим саме вікно міні-гри, і її елементи

Зроблено для того щоб не чекати деякий час на її відпрацювання

ВАЖЛИВА ІНФОРМАЦІЯ ПРО УПРАВЛІННЯ:
- Мова вводу на комп'ютері має бути Англійська (English) обов'язково !!!

-Гра: В стилі Mortal Kombat із двома гравцями {
	1. Управління гравцями {
		1. W або ^ (стрілка вверх) - прижок який працює тільки при бігові!,
		2. A,D або < (стрілка вліво), > (стрілка вправо) - біг в одну із сторін,
		3. S або v (стрілка вниз) - атакування (З обмеженою відстанню)
		4. Додатково {
			1. R - почати новий раунд, при умові якщо у одного із гравців HP опуститься до нуля (або двох)
		}
	},
	2. Закінчення гри {
		1. Один із персонажів виграв - HP (життя) опускається до нуля,
		2. Нічия - закінчується відведений час, двоє помирають
	},
	3. Додатково {
		1. Зміна локації в залежності пересічення однієї із границь карти
	}
}

Додаткова інформація:
 - Пророблено логіку приймання урону гравця з різних сторін, і позицій

Планувалося зробити:
- Зміну різного типу персонажів, і анімацію їхнього появлення/заміну (бажано в іншому вікні під назвою Profile)
  або іншими словами редактор гравця
- Різні типи атак, їх урон, і дальність
- Більше візуальних елементів
- Різного типу локацій і логіку їхньої заміни
- Можливо ще якась ігрова валюта або щось схоже, і досягнення

Пояснення до коду гри

JS Scripts: {
	1. index.js - зберігається основна логіка гри взаємодії, і відмальовкою зображення,
	2. keyboardGuide.js - взаємодія з вікном посібника до клавіатури і виключення/включення анімацій,
	3. gameHistory.js - взаємодія з вікном збереження попередніх ігор гравця,
	4. about.js - взаємодія з вікном 'Про гру' About, а саме: анімацією в залежності ON вона чи OFF
}
Для більш детального, але короткого опису про той чи інший метод постарався коментаріями в коді

СSS files {
	1. index.css - взаємодія стилів з вікном головної сторінки (Play),
	2. keyboardGuide.css - взаємодія стилів з вікном головної сторінки (Keyboard Guide),
	3. gameHistory.css - взаємодія стилів з вікном Game History
	4. about.css - взаємодія стилів з вікном 'Про гру' About
	5. keyframes.css - взаємодія стилів анімацій всіх вікон
}

HTML files {
	1. index.html - головне вікно гри,
	2. gameHistory.html - вікно з записом попередніх ігр,
	3. about.html - вікно 'Про гру' (About)
}

Додаткові папки {
	1. Audio - зберігається аудіо гри,
	2. Images - зберігаються зображення гри (використовуються не всі),
	3. Previews - Зображення вигляду проекту
}
